DigitiZEN

An audio digitization tool.

Digitizen is a simple audio digitizing application.

Its main purpose is very fast preservation of audio materials(up to 16 or more analog machines in parallel), fast search and retrieval via signatures for research and use of such audio materials.

Its telnet based CLI (command line interface) allows for independent recording of preconfigured 8 channels or 4 stereo streams of audio on a custom Linux kernel(or more, theoretically up to 16 or more stereo streams with a special USB audio driver since the CPU isn’t burdened with unnecessary DSP analysis algorithms as the use of such algorithms can be postponed until after the actual digitization had been done and either batch executed or used as plugins in audio restoration software afterwards).

Its tiny source code (under 60kb) makes it easy to study, optimize and adjust it to specific needs.

In 2009 digitizen was able to digitize more than 438 hours of audio, running for several hours each day, during a three months session.

You can view the source code here. Compile with -ljack -lpthread.

Atomic.h was inspired by atomic.h code written by Stephane Letz and peak.c was “stolen” from Ardour and commented upon. Digitizen was written to run as a Jack client.

The checksum output created by digitizen is compatible with cfv which was run after each session in order to make sure that the raw audio data had been correctly recorded. The actual output of digitizen is 8 raw 32-bit normalized float mono streams which can be converted to 24-bit interleaved by a helper program.

The idea was to write a usb audio driver which would output all audio data in 24-bit format directly to digitizen since digitizen checksums all received audio data. A helper program would then batch check the resulting .wav file batch interleaved by another helper program, by deinterleaving and recalculating the checksums of two mono streams, then batch comparing these checksums against digitizen created checksums.
